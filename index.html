Source Code Recommender Systems:
Practitioners' Wishes vs. State-of-the-art

The automatic generation of source code is one of the long-lasting dreams in software engineering research. Several techniques have been proposed to at least speed up the writing of new code. For example, code completion can recommend to developers the next few tokens they are likely to type, while retrieval-based approaches can suggest code snippets relevant for a task at hand. Also, deep learning has been used to automatically generate code statements starting from a natural language description. While research in this field is very active, there is no study investigating what the users of code recommender systems actually need from these tools.

We present two studies involving a total of 131 software practitioners. Study I investigates the characteristics of code recommender systems that are important for practitioners (e.g., the recommended code must use the same coding style of the code under development). The output of this study is a taxonomy of 72 "requirements" that should be considered when designing code recommender systems.
The first round of Study II assesses the practitioners' perceived usefulness of code recommendations generated by four state-of-the-art approaches, studying their strengths and weaknesses and mapping them back to the previously defined taxonomy. We complemented our study with a second round of the Study II that assesses the perceived usefulness of the T5, the best technique from the first round, and Copilot, that reached state-of-the-art performances in code completion task. Our results point to future research directions for code recommenders.
